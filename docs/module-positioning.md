# 模块边界说明

本文档用于说明智能图书馆借阅与座位预约系统的模块边界和职责划分。

## 核心业务模块

### 登录认证

- JWT 作为客户端访问凭证。
- Redis Session 保存服务端登录态。
- `jti` 标识单次登录。
- `tokenVersion` 支持用户状态变化后的旧 token 失效。
- Token 黑名单支持退出登录后的凭证失效。
- 滑动续期用于延长活跃会话有效期。

### 图书查询

- 作为借阅业务入口。
- 支持分页查询、详情查询、分类筛选和热门图书查询。
- 热门图书、图书详情和分类数据使用 Redis 缓存。

### 借阅流程

- 覆盖申请、审核、出借、续借、归还和逾期扫描。
- 使用明确的状态流转表达业务闭环。
- 通过状态校验、条件更新和库存变更保证关键操作一致性。
- 借阅申请使用 Redisson 按 `userId + bookId` 防止重复提交。

### 座位预约

- 覆盖创建预约、时间冲突校验、签到、取消、结束使用和过期处理。
- 创建预约时使用 Redisson 按 `seatId + userId` 加组合锁。
- 锁内完成座位时间冲突和用户时间冲突校验。
- 签到、取消、结束使用通过状态条件更新防止重复操作。

## 工程支撑模块

- Redis 缓存
- Redisson 分布式锁
- AOP 操作日志
- 全局异常处理
- DTO 参数校验
- Spring Schedule 定时任务
- Testcontainers 集成测试
- springdoc-openapi 接口文档

## 基础数据模块

- 图书分类基础数据
- 图书基础数据维护
- 自习室基础数据
- 座位基础数据
- Dashboard overview
- OSS 图片上传

## 已移除链路

- 借阅导出链路已删除。
- 复杂 Report 统计链路已删除，仅保留 Dashboard overview。
- 冗余旧会话服务已删除，认证会话统一收敛到 `SessionManager`。

后续功能扩展应优先围绕当前模块边界演进，避免将统计、导出、文件处理等能力重新耦合到核心业务链路中。
