# 模块定位说明

本文档用于约束项目展示口径：当前项目定位为“智能图书馆借阅与座位预约系统”，不是功能堆叠式后台管理系统。

## 面试主线模块

### 登录认证

- JWT 作为客户端凭证。
- Redis Session 作为服务端登录态。
- `jti` 标识单次登录。
- `tokenVersion` 支持旧 token 失效。
- Token 黑名单支持登出后立即失效。
- 滑动续期提升登录体验。

### 图书查询

- 作为借阅业务入口。
- 保留分页查询、详情、分类筛选和热门图书。
- 热门图书和图书详情缓存体现 Redis 使用场景。

### 借阅流程

- 第一业务主线。
- 覆盖申请、审核、出借、续借、归还和逾期扫描。
- 重点展示状态流转、条件更新、库存一致性和防重复处理。

### 座位预约

- 第二业务主线。
- 覆盖创建预约、时间冲突校验、签到、取消、结束使用和过期处理。
- 重点展示 Redisson 分布式锁和并发冲突控制。

## 工程能力模块

- Redis 缓存
- Redisson 分布式锁
- AOP 操作日志
- 全局异常处理
- DTO 参数校验
- Spring Schedule 定时任务
- Testcontainers 集成测试
- Knife4j / Swagger 接口展示

## 基础支撑模块

这些模块保留，但不作为简历或面试主线：

- 图书分类基础数据
- 图书基础数据维护
- 自习室基础数据
- 座位基础数据
- Dashboard overview
- OSS 基础图片上传
