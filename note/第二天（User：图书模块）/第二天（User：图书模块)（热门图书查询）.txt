1...@RequestParam(value = "limit", defaultValue = "10")

你问：

这个参数怎么传进来的？是前端 JSON 吗？

不是 JSON。

它是 URL 查询参数。

例如前端请求：

GET /book/hot?limit=5

URL 里：

limit=5

Spring 会自动解析：

limit = 5

如果前端不传：

GET /book/hot

就会用默认值：

limit = 10

代码在 Controller 里：


BookController

@GetMapping("/hot")
public Result<List<BookVO>> hot(@RequestParam(value = "limit", defaultValue = "10") int limit)

所以这里 不是 JSON，而是 URL 参数。



2...@Cacheable 是干什么

你说：

不知道在干什么

这是 Spring Cache 注解。

代码在 Service：

BookServiceImpl

@Cacheable(cacheNames = CACHE_BOOK_HOT, key = "#limit")

意思是：

第一次查热门书 → 查数据库
之后相同 limit → 直接从缓存拿

流程：

第一次请求
GET /book/hot?limit=10
        ↓
查数据库
        ↓
结果存入 Redis
        ↓
返回

第二次：

GET /book/hot?limit=10
        ↓
直接从 Redis 返回

好处：

减少数据库查询
提高性能

10...key = "#limit" 里的 # 必须加

如果不加：

key = "limit"

意思就完全不一样了。

下面给你详细讲。

一、#limit 是什么

在 Spring Cache 里：

@Cacheable(cacheNames = CACHE_BOOK_HOT, key = "#limit")

这里的 #limit 是 SpEL 表达式（Spring Expression Language）。

意思是：

取方法参数 limit 的值
作为缓存 key

例如调用：

GET /book/hot?limit=10

方法：

public List<BookVO> listHotBooks(int limit)

参数：

limit = 10

缓存 key 就会变成：

CACHE_BOOK_HOT::10
二、为什么必须加 #

在 SpEL 里：

#变量名

代表：

方法参数变量

例如：

key = "#limit"

表示：

取 limit 参数的值
三、如果不加 # 会发生什么

如果写成：

key = "limit"

那 Spring 会认为：

key 就是字符串 "limit"

所以所有缓存 key 都是：

CACHE_BOOK_HOT::limit

无论你传什么：

/book/hot?limit=5
/book/hot?limit=10
/book/hot?limit=20

缓存都会是：

CACHE_BOOK_HOT::limit

这样就会出现问题：

limit=5 的结果
会被 limit=10 复用

缓存就错了。

四、正确的 key 生成过程

代码：

@Cacheable(cacheNames = CACHE_BOOK_HOT, key = "#limit")

调用：

/book/hot?limit=5

缓存 key：

CACHE_BOOK_HOT::5

调用：

/book/hot?limit=10

缓存 key：

CACHE_BOOK_HOT::10

这样每个参数都有独立缓存。

五、Spring Cache 常见 key 写法
1 参数
key = "#id"
2 多个参数
key = "#userId + ':' + #bookId"

生成：

12:45
3 对象属性
key = "#request.pageNum"
4 默认 key（不写）

如果不写 key：

@Cacheable("book")

Spring 默认用：

所有参数组成 key
六、你这个接口为什么用 #limit

因为方法：

public List<BookVO> listHotBooks(int limit)

唯一影响结果的参数就是：

limit

所以：

key = "#limit"

最合理。
