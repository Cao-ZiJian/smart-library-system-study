SQL 等价：

select count(*)
from borrow_order
where user_id = ?
and book_id = ?
and status in (
  'APPLYING',
  'APPROVED',
  'LENT',
  'OVERDUE'
)

含义：

检查用户是否已经借了这本书

APPLYING   申请中
APPROVED   已批准
LENT       已借出
OVERDUE    逾期

都算 未完成借阅

所以：

就不允许借。

7...public enum BorrowStatusEnum

表示借阅状态枚举，统一管理 borrow_order.status 可取的固定状态值。

APPLYING("APPLYING"), ...

每个枚举常量本质上都是一个 BorrowStatusEnum 对象，并且绑定一个字符串 code。

private final String code;

表示每个状态对象内部都保存一个真正用于业务/数据库存储的状态值。

BorrowStatusEnum(String code)

构造方法，在创建每个枚举对象时把传入的 code 保存到当前对象里。

public String getCode()

用于取出该状态对应的字符串值，方便和数据库中的 status 字段进行赋值、比较、查询。





2次纠正：
1...一、Controller部分

你写：

@RestController json打成string给controll，controll结果打成json给前端

这个理解 有一点点不准确。

真实流程是：

前端JSON
   ↓
@RequestBody
   ↓
Java对象 (BorrowApplyRequest)
   ↓
Controller处理
   ↓
返回 Result
   ↓
Spring自动转JSON
   ↓
返回前端

所以更准确写法：

@RestController
返回对象自动转为JSON返回给前端

不是：

json → string

而是：

json → Java对象
Java对象 → json

@RequestAttribute("currentUserId")

从 request 中读取属性
