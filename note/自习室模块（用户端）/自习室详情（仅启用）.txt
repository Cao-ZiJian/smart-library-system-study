纠正：
一、路径 /study-room/{id} 怎么知道参数名是 id

Controller代码：

@GetMapping("/{id}")
public Result<StudyRoomVO> detail(@PathVariable("id") Long id)

接口路径：

/study-room/{id}

来源：


StudyRoomController

这里的 {id} 叫：

路径变量（Path Variable）

Spring 的规则是：

{变量名}  →  @PathVariable("变量名")

也就是：

{id}  →  @PathVariable("id")
举例

请求：

GET /study-room/5

Spring 会解析：

id = 5

然后传入方法：

detail(5)
如果写成别的名字也可以

例如：

@GetMapping("/{roomId}")
public Result detail(@PathVariable("roomId") Long id)

请求：

/study-room/5

Spring会解析：

roomId = 5

再赋值给参数 id。

二、.one() 有用吗？

你说：

.one() 应该是没用的

其实 非常有用。

代码：

StudyRoom room = lambdaQuery()
        .eq(StudyRoom::getId, id)
        .eq(StudyRoom::getStatus, 1)
        .one();

来源：


StudyRoomServiceImpl

.one() 的意思是：

查询一条记录

对应 SQL：

SELECT * 
FROM study_room
WHERE id = ?
AND status = 1
LIMIT 1
MyBatis-Plus 常见查询方法
方法	作用
one()	查询一条
list()	查询多条
count()	统计数量
page()	分页
为什么要 .one()

因为：

id 是主键

主键特点：

唯一

所以查询结果：

最多一条

使用 .one() 是最合理的。

三、为什么还要加 status = 1

代码：

.eq(StudyRoom::getStatus, 1)

意思：

只允许查询启用的自习室

数据库结构：

status
1 启用
0 禁用

实体类：

private Integer status;

来源：


StudyRoom
